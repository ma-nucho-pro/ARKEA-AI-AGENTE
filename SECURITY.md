# Política de seguridad

No publiques vulnerabilidades con credenciales, datos personales o claves reales en Issues.

Antes de reportar un problema:

- elimina API keys y tokens;
- reemplaza rutas personales por ejemplos;
- no adjuntes bases de datos locales;
- no adjuntes correos ni archivos privados.
